# TTF font rendering (ttf.h)

Proportional bitmap font rendering for HIRES mode, fed by an offline
converter (`tools/oric_ttfconv.py`) that rasterizes a `.ttf` file into a
packed C data table. Include `ttf.h`; it auto-compiles `ttf.c` via
`#pragma compile`, and depends on [hires.h](hires.md) (`HiresBitmap`,
`HiresClip`, `hb_put`).

**Provenance:** the `TtfFont` descriptor shape (fixed height, per-glyph
pixel width, per-glyph byte width, per-glyph offset into a shared glyph
blob, plus alignment-aware print helpers) is inspired by raxiss/iss000's
[lib-ttf](https://github.com/iss000/oricOpenLibrary/tree/main/lib-ttf) —
credit for the *concept*, not a port. lib-ttf's own runtime is hand-written
6502 assembly fed by a Lua + ImageMagick offline pipeline; this is plain
Oscar64 C reusing `hires.c`'s `hb_put()`-based per-pixel rendering (the
same approach as `hb_put_chars()`), fed by a pure Python + Pillow
converter. The packed glyph-data format is this project's own convention
(6px/byte, bit5=leftmost, local to each glyph), not lib-ttf's assembly data
layout, and `ttf_print` takes normal C parameters rather than lib-ttf's
global-state macro calling convention (`ttf_open`/`_ttf_x`/`_ttf_y`).

## Converting a font: `tools/oric_ttfconv.py`

```
python3 tools/oric_ttfconv.py FONT.ttf POINTSIZE OUTPUT.h [--label NAME] [--first-char N] [--last-char N]
```

Requires Pillow (`pip install -r tools/requirements.txt`, shared with
`oric_pictconv.py`). Rasterizes every character in `[first-char, last-char]`
(default: printable ASCII 32-126, matching `charset.h`'s ROM range) via
Pillow's `ImageFont.truetype`/`ImageDraw`, using the font's own advance
width (`font.getlength()`) as each glyph's pixel width (so text keeps the
font's natural proportional spacing) and the font's ascent+descent
(`font.getmetrics()`) as a common height shared by every glyph, so all
characters render on a consistent baseline.

Output is a `.h` file defining `{label}_widths[]`, `{label}_width_bytes[]`,
`{label}_offsets[]`, `{label}_char_defs[]`, and a `const TtfFont {label}`
initializer wiring them together — `#include` it directly and pass
`&{label}` (or just `{label}` where a `const TtfFont *` is expected) to the
`ttf_*` functions below.

## `TtfFont` — font descriptor

```c
typedef struct {
    uint8_t w;                     // max glyph width in pixels (informational)
    uint8_t h;                     // glyph height in pixels (rows), same for every glyph
    uint8_t first_char, last_char; // ASCII code range covered
    const uint8_t *widths;         // per-glyph pixel width, indexed by (code - first_char)
    const uint8_t *width_bytes;    // per-glyph byte width = ceil(width/6)
    const uint16_t *offsets;       // per-glyph byte offset into char_defs
    const uint8_t *char_defs;      // packed glyph rows, 6px/byte, bit5=leftmost (glyph-local)
} TtfFont;
```

Glyph rows in `char_defs` are packed the same 6-pixels-per-byte way as a
HIRES pixel byte's low 6 bits, but **without** bit6/bit7 — those are added
at render time by `ttf_print`, not baked into the converted data, since
this is glyph-local pixel data, not a HIRES screen byte.

## Functions

```c
extern uint8_t ttf_space;   // extra inter-character spacing in pixels (default 1)
```

```c
uint16_t ttf_strlen(const TtfFont *font, const char *s);
```
Total rendered pixel width of a string under the given font (sum of glyph
widths + `ttf_space` between characters). Characters outside
`[first_char, last_char]` are skipped (zero width) — no error, no
placeholder glyph.

```c
uint16_t ttf_print(const HiresBitmap *hb, const HiresClip *clip, const TtfFont *font, uint8_t x, uint8_t y, const char *s);
```
Render a string at pixel position `(x, y)`. Returns the pixel width
consumed.

```c
uint16_t ttf_print_left(const HiresBitmap *hb, const HiresClip *clip, const TtfFont *font, uint8_t y, const char *s);
uint16_t ttf_print_center(const HiresBitmap *hb, const HiresClip *clip, const TtfFont *font, uint8_t y, const char *s);
uint16_t ttf_print_right(const HiresBitmap *hb, const HiresClip *clip, const TtfFont *font, uint8_t y, const char *s);
```
Convenience alignment helpers over `ttf_print`/`ttf_strlen` — left/centre/
right align within the full `HIRES_WIDTH_PX`-wide screen (not a caller-
supplied width).

## Example

```c
#include "hires.h"
#include "ttf.h"
#include "myfont.h"   // generated by oric_ttfconv.py, defines `const TtfFont myfont`

HiresBitmap hb;
hb_init(&hb, (uint8_t *)HIRESVRAM, HIRES_ROWS);
hb_fill(&hb, 0x40);

ttf_print_center(&hb, (const HiresClip *)0, &myfont, 20, "ORIC DEMO 2026");
```
